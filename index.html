三、三 Tier 動態流水號規則

《2026 報單號碼規則 Excel》明訂流水號（pos10-14）依報單類別分三層嚴格度。新系統按 declarationType 動態決定所屬 Tier，並配合 transportMode（海 / 空）決定 charset 與 pos10 策略。

Tier A 最嚴 — 空運純數字 5 碼（首位按站）/ 海運 pos10 鎖收單首字 + 4 英數 + A72 查重

適用類別：F2 / F3 / D2 / D7（本體）+ B6 / D8（自由港版）

3.7 海運站別辨識（補償方案）

海運 pos10 被 Excel 規則 / OP 慣例佔用（Tier A 收單首字 / Tier B 預設 E/I / Tier C 按類別預設），無空間放站別。補償三件組：
  1. Entity 欄位：CustomsDeclaration.createdByStation: StationCode（打單 OP 所屬站自動帶入）
  2. 列印檔名尾碼：{declNo}-{station}.{ext}，例：CWD2151420579-H.pdf（高雄 = H）
  3. 報單右上角 badge：視覺辨識 🏢 新竹 / 🏢 桃園 / …
B1 / B2 / G2 站別靠客戶主檔綁定（保稅廠客戶 = 固定站，聯電永遠新竹、台塑永遠高雄等），不需流水號帶。

四、A72 跨類別查重

財政部關務署 114.6.9 修正的不受理原因代碼 A72：自由貿易港區報單（F2 / F3 / B6 FTZ / D8 FTZ）第 5-14 碼不可重複。LRP 原始碼 lcGet_4() 有實作此邏輯，新系統沿用並強化。

4.1 為什麼需要跨類別查重

A72 限制是 跨類別的：即便 F2 跟 B6 FTZ 是不同報單類別，若兩者的 pos5-14 相同仍會被海關拒收。LRP 的解法是：

// LRP/prodecl/PUB_FUNC.pas lcGet_4() SELECT COUNT(*) FROM SCUSDEC WHERE SUBSTR(DECL_NO, 5, 10) = '15012E0001' AND DECL_TYPE IN ('F2','F3','B6','D8') // Tier A 自由港子集 AND WAREHSE_YN = 'Y'; // 僅 FTZ 版 B6/D8

async function checkCollision( declType: DeclarationType, pos5to14: string, isFreeTradeZone: boolean, ): Promise<boolean> { if (!needsA72Check(declType, isFreeTradeZone)) return false; // 查自由港區子集內是否已存在相同 pos5-14 const exists = await this.dataSource .createQueryBuilder() .from(CustomsDeclaration, 'd') .where('SUBSTR(d.decl_no, 5, 10) = :key', { key: pos5to14 }) .andWhere('d.declaration_type IN (:...types)', { types: ['F2', 'F3', 'B6_FTZ', 'D8_FTZ'], }) .getCount(); return exists > 0; }

5.4 CustomerBondedBranch entity 職責

  • OP 打報單時，選擇買方 → 系統從 CustomerBondedBranch join 找對應 bfno → buildDeclNoPos7to10() 自動組字
  • 出口 / 進口方向不同（B1/B2 出口 vs G2 進口），自動對照正確補字方向
